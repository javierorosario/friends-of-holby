# Mentor Review Checklist  
*Mentored Real-World Software Project*

> This checklist is for **mentor use only**.  
> The goal is to evaluate **process, clarity, and growth** — not just code quality.

---

## 1. Project Framing Review

☐ Problem statement is clear and understandable  
☐ Target user is clearly identified  
☐ Success criteria are concrete and realistic  
☐ Non-goals are explicitly stated  
☐ Scope is appropriate for team size and timebox  

**Notes:**

---

## 2. Requirements & Backlog Quality

☐ User stories are written from a user perspective  
☐ Acceptance criteria are present and testable  
☐ Stories are small enough to complete in a sprint  
☐ Prioritization makes sense (Must / Should / Could)  
☐ No premature technical optimization in requirements  

**Red flags to watch for:**
- Stories are technical tasks only
- “Do everything” scope
- Missing acceptance criteria

## 3. GitHub Project Usage

☐ GitHub Project board is actively used  
☐ Issues move across columns intentionally  
☐ Sprint Backlog is respected  
☐ Issues are linked to Pull Requests  
☐ Board reflects real project state (not aspirational)

**Notes:**

---

## 4. Git & Pull Request Discipline

☐ Work is done in feature branches  
☐ Pull Requests are small and focused  
☐ PR descriptions explain *why*, not just *what*  
☐ PRs link to issues (`Fixes #`)  
☐ Feedback is responded to constructively  

**Red flags to watch for:**
- Large “everything” PRs
- Merging without review
- Defensive responses to feedback

**Notes:**

---

## 5. Communication & Collaboration

☐ Mentee asks clarifying questions early  
☐ Trade-offs are explained when needed  
☐ Status updates are clear and honest  
☐ Blockers are raised promptly  
☐ Communication stays professional and respectful
